Arquitectura

n8n orquesta. La voz conversa. Los datos quedan auditables.

La demo está diseñada para sobrevivir fallas de red: Supabase primero, JSON fallback como última defensa.

01

Cache de datos

Oportunidades y perfiles se leen desde Supabase o `fallback.json` con estructura equivalente.

02

Matching por rubro

Rubro exacto sube el score a 100. Coincidencias parciales quedan como recomendación secundaria.

03

Llamada con voz

n8n envía variables dinámicas a ElevenLabs: productor, rubro y oportunidad detectada.

04

Preguntas duras

Codex/OpenAI responde con preguntas, intención y palabras clave para guiar la conversación.
