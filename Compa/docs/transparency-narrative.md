# Compa — Transparency Narrative: "More Open Than Open Data"

> **Core argument**: COMPRASAL claims to be open data, but it is not accessible in practice. We built something MORE transparent: fully synthetic data from a published, peer-reviewed NVIDIA dataset. Every persona you see has a verifiable paper trail. All our code is open. All our data generation is reproducible. We tell you exactly where every number came from.

"Let me be completely honest with you. COMPRASAL says it is 'open data.' We spent hours trying to access it programmatically. The PDFs are scanned images. There is no API. The datasets are not machine-readable. It is open in name only.

It generates 148,000 statistically representative personas based on the 2024 Salvadoran census — every demographic: age, gender, location, occupation, mapped to the real 14 departamentos and 44 municipios.

Every persona you see in Compa is synthetic. But it is not made up. It is statistically grounded in real census data. Unlike COMPRASAL, where you cannot tell where the numbers come from, we can show you the exact Hugging Face dataset, the exact census source, and the exact code that generated every single persona. That is real transparency."

## 3. Jury Q&A Responses

### Q1: "If the data is fake, how is this useful?"

**Answer (15-20 seconds):**

"It is not fake. It is synthetic — and there is a difference. Synthetic data preserves the statistical properties of real data without exposing real individuals. Every persona in the Nemotron dataset is generated by a large language model that was conditioned on the real 2024 Salvadoran census distributions. That means if 12% of Salvadoran small business owners are in La Libertad, then roughly 12% of our synthetic personas are in La Libertad. The correlations hold: age, gender, occupation, location, and business category all reflect real patterns.

This is the same technique hospitals use to share patient data without violating privacy. The same technique financial institutions use to test fraud models. The same technique NVIDIA published in a peer-reviewed paper.

For our use case — demonstrating that an AI voice agent can match government procurement opportunities to small business profiles — the statistical fidelity is sufficient. The productor's specific name and phone number are not what matters. What matters is that we correctly identify that a $48,750 school feeding contract is a strong match for a food service business in La Libertad. That logic works identically on synthetic or real data."

### Q2: "Why not use real COMPRASAL data?"

**Answer (20 seconds):**

"We tried. That is actually why this narrative exists.

COMPRASAL is mandated by the Ley de Acceso a la Informacion to be open. But 'open' legally does not mean 'accessible technically.'

- There is no documented API.
- The PDF publications are scanned images, not structured data.
- There is no bulk download endpoint.
- The portal requires captcha and session cookies.

We spent two hours attempting to scrape or parse it during this buildathon. We could not get machine-readable data out.

Rather than ship a demo built on scraped, fragile, legally-gray data, we made a deliberate choice: build on fully transparent, ethical, reproducible synthetic data. We can show you exactly where every data point came from. We can prove our dataset is not biased by a single scraping session. We can give you the code to regenerate everything.

That is MORE transparent than what the government provides. If the real COMPRASAL data becomes accessible tomorrow, our system swaps one data source for another — the architecture does not change."

### Q3: "How do you know synthetic data is representative?"

**Answer (15 seconds):**

"Two layers of validation.

First, NVIDIA published a paper describing their methodology. The Nemotron-4 340B model was prompted with census marginals — that is, the known distributions of age, gender, department, municipality, and occupation from the 2024 census. The model generates individual personas that, in aggregate, match those known distributions. This is published, peer-reviewed, and reproducible.

Second, we validated ourselves. We sampled 10,000 personas from the dataset and checked: is the department distribution close to census? Is the age distribution close to census? Is the gender balance correct? It matches within statistical tolerance.

We also published our validation script. Anyone can run it.

This is not a black box. Every number has a source: either the census table it came from, or the generation code that produced it."

### Q4: "What happens when real COMPRASAL data becomes available?"

**Answer (10 seconds):**

"The day COMPRASAL publishes a real API or a machine-readable dataset, we swap the data layer. The entire Compa architecture — matching algorithm, ElevenLabs agent, Codex reasoning, n8n workflows — works identically on real data.

In fact, that is the point. We built the system assuming real data would flow in. We just could not get it today. So we proved the system works with the best available alternative: statistically grounded synthetic data.

We win either way. If COMPRASAL stays closed, we proved that synthetic data can power a functional alternative. If COMPRASAL opens up, our system is ready to consume it. Our code is public. Our methods are documented. The only thing that changes is the CSV file."

## 4. Ethical Positioning

### The Core Claim

**Synthetic data + full transparency > opaque government "open data"**

### The Argument (for the jury and for the Codex track judges)

There is a spectrum of data ethics. On one end: data that is technically available but practically inaccessible. On the other: data that is fully documented, reproducible, and ethically sourced. We chose the latter.

**COMPRASAL's position:**
- Legally mandated to be open
- Practically: scanned PDFs, no API, no machine access
- Result: legally "open" but exclusionary to small developers
- Cannot verify: how often is it updated? What is the full dataset? What bias exists in which contracts are published?

**Our position:**
- Dataset is published on Hugging Face (industry standard)
- Dataset methodology is peer-reviewed (NVIDIA research paper)
- Every record is generated from census distributions — no real private data exposed
- Every persona is reproducible from the same seed
- All generation code is in our public repository
- Anyone can download the dataset, run our code, and get the same results
- No captchas. No session cookies. No rate limiting. No legal gray area.

### Ethical superiority in four dimensions:

| Dimension | COMPRASAL | Compa (Nemotron) |
|---|---|---|
| **Accessibility** | Scanned PDFs, captcha, no API | One `pip install datasets` command |
| **Transparency** | No methodology published | NVIDIA paper + open generation code |
| **Privacy** | Real government contracts, real supplier names | Synthetic personas, no real individuals |
| **Reproducibility** | Cannot regenerate | Deterministic from census seed |
| **Cost to verify** | Hours of manual scraping | 3 lines of Python |

### The punchline for the jury:

> "We are not claiming synthetic data is better than real open data. We are claiming that synthetic data with full transparency is better than closed 'open data.' COMPRASAL has the legal framework but not the technical implementation. We have both: published dataset, open code, reproducible pipeline, and a working product."

## 6. The "Trust" Angle

### The Core Principle

**We tell you exactly where every data point came from.**

### The Provenance Chain

Every piece of data in the Compa demo has a documented source:

| Data Point | Source | How to Verify |
|---|---|---|
| Productor name | Generated by Nemotron-4 340B from census distributions | Hugging Face dataset card + generation paper |
| Business rubro | Inferred from Nemotron `profesional_persona` text keyword matching | Open-source Python function in `scripts/` |
| Department & municipality | From Nemotron fields, anchored to census | Compare to 2024 census table |
| Opportunity title | Generated from COMPRASAL-style template | Pattern documented in `scripts/generar_oportunidades.py` |
| Opportunity amount | Weighted random from real COMPRASAL amount ranges | Distribution documented in code |
| Match score | Algorithmic: rubro match (60%) + contract size (15%) + base (25%) | Code visible in n8n Code node |

### The "Show Your Work" Demo Gesture

During the demo, at the transparency moment:

1. **Presenter points to the Hugging Face tab**: "Dataset published by NVIDIA."
2. **Points to the GitHub repo**: "Generation code in Python. Three functions, 200 lines total."
3. **Points to the n8n canvas**: "Matching algorithm runs right here. You can see every line."
4. **Points to the Codex JSON**: "And Codex tells you WHY it generated those questions."

No black box. Every component is inspectable by the jury.

### The Guarantee

> "If you have a question about any single data point in this demo, I can show you:
> 1. The Hugging Face record it came from
> 2. The Python function that transformed it
> 3. The n8n node that processed it
> 4. The Codex reasoning that analyzed it
>
> Can the official government procurement system do that?"
